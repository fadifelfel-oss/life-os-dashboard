## ❌ What needs attention

### 1. Hermes cannot ACT. This is the whole ballgame. 🔴
`grep -c "tool_call\|tools=\|function_call" server.py` → **0**.
`/api/chat` (`server.py:3361`) is a blind passthrough proxy to the Hermes API Server at `127.0.0.1:8642`.
Text in, text out. There is no tool layer, no function registry, no action dispatch.

**Consequence:** today you can *ask* Hermes to add a task. He will reply "Sure, I've added it." Nothing
will be added. Every EA action you listed — to-do, email draft, show me a file, schedule a calendar —
is not possible in the current architecture. This is not a bug; the tool layer was never built.

## 📞 Voice / phone: yes, and here's the honest version

**What you have:** one-way, record-then-send speech→text on the desktop.
**What "talk to it like Gemini" needs:** secure origin → mic → STT → **agent with tools** → TTS → audio out.

| Layer | State | Path |
|---|---|---|
| Secure origin | ❌ missing | Caddy + a domain. **Gate for everything else.** |
| Mic capture | ✅ built (`shared.js:1327`) | Works once HTTPS lands. |
| STT | ✅ built (`/api/transcribe`, faster-whisper) | Consider upgrading `base` → `small` for Arabic. |
| **Agent with tools** | ❌ **missing** | The real work. See item 1. |
| TTS (Hermes talks back) | ❌ missing | Browser `speechSynthesis` — **free, ~20 lines, zero backend.** |
| Phone access | ❌ missing | **PWA** ("Add to Home Screen") — no app store, no cost. |
| Real-time duplex | ❌ missing | OpenAI Realtime / Gemini Live. WebSocket rework. Later. |
| Actual phone *call* (dial a number) | ❌ | Twilio. Real money + real complexity. **Not recommended now.** |

**Verdict:** tap-to-talk on your phone, Hermes speaks back, and he can actually do things — that is
**achievable and gets ~80% of the Gemini feel.** Real-time interruption-capable duplex is a later horizon.

**Sequencing that matters:** voice before tool-calling gives you a talking chatbot that still can't add a
task. **Tools first. Voice second.**

### How Hermes gets Gmail/Calendar hands without you writing integrations

You already pay for **Make.com through April 2027**. Recommended pattern:

```
Hermes  →  tool_call {name: "draft_email", args: {...}}
        →  server.py dispatches to a Make.com webhook
        →  Make does the Gmail draft / Calendar event / Drive lookup
        →  result returns to Hermes  →  Hermes tells you it's done
```

One webhook per action. You build them click-by-click in Make, no code. This is the lowest-effort route
from your actual stack to a working EA, and it keeps credentials out of `server.py`.
