Loops — the whole robot workforce

Every scheduled job across Cowork, the VPS, and Windows Task Scheduler, in one place. Rule: every new automation gets a row here before it's created — no orphan crons.

Trigger→ Signal→ Action→ Eval→ Stop
Division of labor. Cowork scheduled tasks own anything that touches the vault's content or Notion (they run under agents.md with connectors). Hermes / VPS crons own anything that touches the server (pull, restart, mirror). Windows Task Scheduler owns anything that must run even when the Claude app is closed (git pushes of local folders). Never give two runners the same job.
