Your AIOS runs the Nous hermes-agent framework on your VPS, calling models through OpenRouter. Hermes is the always-on "hands" that pulls from GitHub, deploys, and ingests new clippings. Its powers come from the tool layer you wire it with — not the model weights — so design the tool set deliberately.

Which model to run

Hermes 3 (Llama-3.1 8B/70B/405B) and Hermes 4 (70B/405B) are natively agentic: they emit <tool_call> JSON to self-select tools, carry 128K context, and hold JSON-schema structured output. On tool-use benchmarks they track GPT-4 / Claude-3-Sonnet.

  • Hermes 4 70B — default for most agent work: fast, cheap, reliable tool-calling.
  • Hermes 4 405B — reserve for the hardest reasoning/planning steps.
  • Hermes 4 adds a toggleable <think> reasoning mode — turn it on for planning, off for speed.

Tool / function calling

Hermes decides when a tool is needed and returns a JSON <tool_call>. Design for reliability:

  • Keep each tool atomic and narrow (one job, clear args) — easier for the model to call correctly.
  • Give every tool a strict JSON schema; validate the output before executing.
  • Gate destructive actions (deploy, delete, git force-push) behind an explicit confirm step.
  • Return short, structured tool results — long dumps waste context.

OpenRouter routing

Route by job, not habit — this is where cost is won or lost:

  • Cheap/fast models for ingestion & summarizing (high volume).
  • Reasoning models only for analysis & opportunity-spotting.
  • Set provider fallbacks so a busy provider doesn't stall the agent.
  • Use OpenRouter's price/latency routing preferences to cap spend.

See the By Job tab for the exact model to route each task to.

Cost & reliability discipline

  • Log every call in the Token Tracker tab — you can't optimize spend you don't measure.
  • Cache/skip re-processing clippings already ingested.
  • Prefer smaller models with a review step over a big model with none.
  • For anything that writes to your repo or VPS, keep a human-in-the-loop confirm — the deploy pipeline is Claude edits → GitHub → Hermes pulls.
